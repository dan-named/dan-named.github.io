Ключевые моменты:

Автономная работа за период между сессиями:
В период без созвонов с Даном боты у Кати падали несколько раз. Она сама поднимала их через терминал, потом писала боту разобраться в причинах падения, зафиксировать в документации, чтобы такие случаи больше не повторялись. Это была первая полностью автономная отладка без помощи извне.

--dangerously-skip-permissions:
Бот периодически продолжает спрашивать подтверждения у Кати на отдельные действия (обычно при редактировании системных файлов вроде CLAUDE.md). Дан напомнил про команду --dangerously-skip-permissions для запуска Claude Code без подтверждений. Команду наизусть помнить не нужно — найдётся в документации за минуту.

Использование Claude из бота на ходу:
Катя подтвердила, что её основной use case — обращаться к боту прямо с телефона, не открывая поисковики. Все вопросы идут сразу к Claude. Это снижает когнитивную нагрузку и ускоряет принятие решений.

Кейс защиты от мошенников:
К Кате как продавцу эфирных масел doTERRA написал «клиент» с просьбой созвониться. После двух подозрительных Zoom/Google Meet ссылок (одна не пустила в звонок, другая просила вводить пароль заново) Катя отправила одну из ссылок в бота. Claude распознал фишинг, описал признаки и подсказал шаги защиты — почистить браузер и историю, мониторить активность на банковских картах. Дан попросил разрешения упомянуть кейс в публичном канале — Катя согласилась прислать скриншот ответа бота.

Multi-agent внутри одной Claude Code сессии:
Дан рассказал концептуальный сдвиг: вместо настройки межботной делегации (когда один Telegram-бот пишет другому через общий чат), внутри одной Claude Code сессии можно запустить неограниченное число локальных агентов через Task tool — десять, тридцать, сколько нужно. Издержки минимальны, общение между ними не требуется. Совет на следующую комплексную задачу: вывалить всё неструктурированное и сказать «запусти процессы» — Claude сам распараллелит работу на анализ, планирование, настройку, документацию.

VPS reboot через Digital Ocean:
На случай, когда бот лежит и терминал не запускается — заходить в браузерный аккаунт Digital Ocean и перезапускать VPS как обычный компьютер (выключить/включить). Это крайняя мера, но безопасный путь восстановления, когда Termius не помогает.

Документирование протоколов вместо ad-hoc запоминаний:
Катя спросила: каждый раз после нового шаблона работы говорить боту «запомни, зафиксируй»? Дан ответил, что лучше системно — описывать протоколы в отдельных файлах, а в CLAUDE.md прописывать триггеры вида «когда я пишу X — читай файл Y». Тогда косяки не повторяются и не нужно объяснять заново при каждой сессии.

Совместная работа с Максом через GitHub:
Макс написал Кате накануне, попросил подготовить вопросы для совместной работы через GitHub. Катя подтвердила, что хочет настраивать GitHub-флоу вместе с Максом и Даном на следующей встрече, когда накопятся конкретные вопросы. Дан подвёл итог: Катя стала достаточно автономным бойцом и готова к командной работе.
