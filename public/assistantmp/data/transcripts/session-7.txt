Ключевые моменты:

Настройка транскрипции:
Первые полчаса ушли на настройку инструмента транскрипции. Нужно было переключить язык распознавания с английского на русский. Катя настраивала доступ с нового устройства. Транскрипт записывается автоматически и сохраняется в текстовый файл — это первый транскрипт с девайса Кати.

Переавторизация Claude Code на VPS:
При подключении к VPS обнаружилось, что токен Claude Code истёк (token has expired). Прошли повторную авторизацию через /login — браузер открывается, подтверждается подписка. Катя уже делала это раньше, процесс знакомый. Дан подчеркнул: переавторизация — рутинная операция, будет повторяться периодически.

Telegram-бот — реакции и память:
Бот ставит эмодзи-реакции на сообщения (глаза 👀 = смотрю, лайк = сделано). Для Кати это было неожиданностью — она думала, что сама случайно нажала. Обнаружили, что бот не сохраняет историю переписки между сессиями: каждый новый чат начинается с чистого листа. Это ограничение текущей архитектуры — для памяти между сессиями нужна отдельная задача (summary-файл, подгрузка при старте).

Архитектура VPS — ревью и чистка:
Попросили Claude Code описать текущую архитектуру на VPS. Обнаружили: активен старый кастомный бот, а telegram-плагин для Claude Code установлен, но не запущен — это другой подход, где Claude Code сам слушает Telegram напрямую. Решили почистить: удалить ненужные компоненты безопасно, не ломая рабочую конфигурацию. Claude Code составил план, Катя попросила review плана перед исполнением.

Watchdog — поддержание сессии:
Дан объяснил концепцию watchdog-скрипта: tmux-сессия с Claude Code может уйти в сон, если не писать. Решение — cron-задача, которая каждые 2 минуты проверяет, жива ли сессия, и пингует соединение. Катя записала термины: watchdog, cron. Подход: описать задачу простым языком без технических терминов — Claude Code сам подберёт реализацию.

Google Tasks API:
Катя хотела создавать задачи (tasks) через бота. Бот ответил, что может создавать события на весь день в Calendar, но настоящие Google Tasks — отдельный сервис с отдельным API. Нужно зайти в Google Cloud Console и сделать enable Google Tasks API в рамках текущего приложения (того же, где Calendar API). Аналогия: Google Sheets API — для таблиц, Google Drive API — для комментариев. Экосистема связана, но API разделены.

Документация изменений:
Дан показал подход: после изменений на VPS говорить Claude Code «составь план документации последних изменений, которые не отражены». Агент составляет план, согласовывает, затем обновляет документацию. Важно: сначала план, потом исполнение. Claude Code предложил два варианта — оставить как инструкцию или заменить VPS-специфичным. Катя сказала: распределяй по папкам и документируй параллельно в Obsidian-хранилище.

Демо — Telegram-канал Дана:
Дан показал свой Telegram-канал (~500 подписчиков), где рассказывает про AI. Катя увидела посты про эмодзи-реакции бота — совпало с тем, что обнаружили на уроке.
