Основная тема: Установка Claude Code на MacBook

Катя попыталась установить Claude Code самостоятельно до сессии, но столкнулась с ошибкой "command not found" — пыталась вспомнить команду по памяти вместо того, чтобы загуглить.

Дан объяснил ключевой принцип: ничего не нужно запоминать. Всё знает интернет, всё знает AI, и всё можно задокументировать. Голову освобождаем на уровень стратегического видения, а всё остальное забирают AI.

Катя нашла установочную команду через Google, но получила ошибку "could not resolve host". Скопировала ошибку в Google AI (режим AI Overview), получила диагностику — проблема с DNS/доступом к серверу из России.

Шаги решения:
1. Проверка DNS — работает исправно
2. Полный сброс сетевого кэша (потребовался пароль от компьютера)
3. Смена VPN-локации на США (Германия не помогла)
4. Повторный запуск установки — успех

Параллельно обсудили:
- Как перекидывать информацию между AI-агентами (копировать ответ одного в другой)
- Universal Clipboard между iPhone и Mac для обхода VPN-конфликтов
- День весеннего равноденствия и балийский Новый год (день тишины)
- Дан рассказал про свой Telegram-бот для управления задачами и календарём через AI
- Цифровой дневник как контекст для AI — чем больше загружаешь, тем лучше помощь
- Философия: сейчас нет экспертов в AI-бизнесе, есть только энтузиасты

Итог: Claude Code установлен, аккаунт на claude.ai оплачен. Следующий шаг — подключиться через терминал и начать общаться.
