Meeting Title: Workflow optimization for task management and automation tools
Date: Apr 6, 2026
Meeting participants: Dan V., Ekaterina

Темы сессии:
- Google Tasks vs TickTick — выбор единого источника задач
- Планирование дня через агента — задачи + календарь в одном сообщении
- Context switching — минимизация переключений между приложениями
- Cron-расписание — автоматическая утренняя сводка в Telegram
- Команда /focus — кастомная slash-команда в Telegram-боте
- Кнопки меню Telegram-бота — настройка UI для быстрого доступа
- Методология спринтов для агентов — коллаборация через GitHub и статус-файлы
- Голосовые сообщения — транскрипция через OpenAI Whisper API
- Документация процессов — way of work в Obsidian

Ключевые моменты:

Google Tasks vs TickTick — выбор одного инструмента:
Катя хотела подключить Google Tasks API, но Дан предложил не вести два источника задач параллельно. Аргумент: агент должен однозначно понимать, что «создай задачу» = TickTick, без дополнительных уточнений. Два инструмента создают рассинхрон и лишнюю коммуникацию без добавленной ценности. Катя согласилась отказаться от Google Tasks в пользу TickTick.

Планирование дня через агента:
Катя привыкла видеть задачи рядом с календарём в Google Tasks. Дан предложил альтернативу: агент с доступом к TickTick и Google Calendar может составлять расписание на день, распределяя задачи по временным слотам. Катя выразила опасение, что её задачи (в основном коммуникации) непредсказуемы по времени. Дан предложил решение: просто спрашивать агента «что я ещё не закрыла сегодня?» — быстрее и удобнее, чем сверяться по приложениям.

Context switching и единый интерфейс:
Дан поделился наблюдением с лекции: переключение между приложениями (Telegram, Google Calendar, TickTick) — это context switching, который можно устранить, общаясь с агентом как единой точкой входа. При этом подчеркнул разницу: автоматизация командных процессов через агента — да, но human touch в личной коммуникации лучше сохранять.

Утренняя сводка по cron:
Настроили автоматическую отправку плана дня в Telegram каждое утро в 8:00 (GMT+3, Краснодар). План включает события из Google Calendar и активные задачи из TickTick. Катя получила первое сообщение и начала кастомизировать формат: задачи на сегодня + события на несколько дней вперёд.

Команда /focus в Telegram-боте:
Создали кастомную команду /focus для получения актуальной сводки по запросу (не только утром). Возникла путаница между slash-командами Claude Code в терминале и slash-командами Telegram-бота — агент на VPS воспринял команду как внутреннюю. Уточнили контекст, команда заработала. Кнопка появилась не в меню, а как inline-клавиатура внизу чата.

Методология спринтов для агентов:
Дан рассказал про методологию совместной разработки через агентов: sprint collect (сбор задач), sprint run (запуск с проверкой, не работает ли другой участник), статус-файл для сигнализации активного спринта. Всё реализовано текстовыми инструкциями в файлах. Пообещал прислать Кате набор файлов как стартовый шаблон.

Автоматизация записей Zoom:
Дан привёл пример своей автоматизации: система сканирует Google Calendar каждые 30 минут, находит завершённые Zoom-записи по тегу, загружает на YouTube с автоматически сгенерированными описаниями, таймкодами и обложкой, а затем добавляет на LMS-платформу. Всё на входе требует только ID Google Calendar.

Голосовые сообщения в боте:
Катя спросила про голосовой ввод. Дан объяснил: у Claude нет speech-to-text API, используется OpenAI Whisper (~$5/мес). Для реализации нужно форкнуть официальный Telegram-плагин Claude Code и встроить транскрипцию голосовых сообщений. Альтернатива — транскрибировать на телефоне и отправлять текст.

Документация и way of work:
Дан подчеркнул важность документирования решений: как работает утренняя сводка, какие приоритеты, какой формат. Это защищает от случайного затирания настроек агентом. Катя вспомнила, что структура папок в Obsidian уже была создана ранее. Дан рекомендовал перед документированием всегда просить агента сначала посмотреть на существующую структуру и предложить, куда лучше записать.
