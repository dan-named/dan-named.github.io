Ключевые моменты:

Frontend vs Backend Telegram-бота:
Катя создала бота в BotFather и получила токен — это frontend. Но без backend на сервере бот не отвечает на сообщения. Claude Code предлагал обойтись без VPS (напрямую через Anthropic API из Python), но Дан объяснил: Python-скрипт всё равно должен где-то работать 24/7. На локальном компьютере бот умрёт при закрытии крышки. VPS решает эту проблему. Anthropic API при этом никуда не девается — он будет использоваться внутри бота на VPS.

SSH-ключи вместо пароля:
На прошлой сессии подключались к VPS по паролю. Сегодня настроили SSH-ключи: зашифрованный ключ на Mac + соответствующий на VPS. Теперь подключение без ввода пароля. Claude Code сам сгенерировал ключи и скопировал на сервер. Были проблемы с подключением (сервис перезагружался), но после повторной попытки всё заработало.

Два терминала — новый режим работы:
Катя перешла в режим работы с двумя терминалами одновременно. Первый — Claude Code на VPS (через SSH). Второй — Claude Code на Mac для отладки и помощи. Если в одном терминале агент заходит в тупик и начинает ходить по кругу (залипает на ошибочных путях в контексте), можно попросить summary проблемы, перенести в другой терминал для свежего взгляда и вернуться с решением. Это «breakthrough»-техника.

Claude Code на VPS с dangerously-skip-permissions:
Запустили Claude Code из папки Obsidian-хранилища на VPS в режиме полных разрешений. Перед этим разобрались с навигацией: команда cd ~/path/to/obsidian (на Linux без дефиса перед тильдой, в отличие от Mac). Катя отправила инструкцию по настройке бота с пояснением конечной цели — Дан подчеркнул важность декларации намерения (образа результата), а не только технических шагов, для повышения точности работы агента.

Telegram-бот — первый успешный запуск:
Бот заработал — отвечает интерактивно с AI на VPS. Катя отправила сообщение из Telegram, бот обработал и ответил. Обсудили user ID vs token: бот принимает сообщения от любого аккаунта, но на уровне VPS можно настроить фильтрацию по ID для безопасности.

Перенос авторизаций с Mac на VPS:
На Mac настроены интеграции (TickTick, Google Calendar), но на VPS их нет. Попробовали перенести credentials напрямую. Claude Code на VPS предупреждал о небезопасности — Катя подтвердила, что это её VPS и её авторизации. TickTick-токен истёк — нужна переавторизация. Дан порекомендовал запросить у Claude Code способ сделать долгоживущий токен, чтобы не повторять авторизацию.

Dropbox-синхронизация — отложена:
До настройки Dropbox не добрались. Дан пообещал сделать review своей настройки синхронизации Obsidian между Mac и VPS и прислать инструкцию.

Рост самостоятельности:
Дан отметил, что Катя уже близка к самостоятельной работе. Основной паттерн: декларировать желаемый результат → Claude Code ищет техническое решение → если ограничение — просить найти обход. Катя выступает менеджером со своими хотелками, Claude Code — исполнителем в рамках технических возможностей. Дан: «Мы уже очень близко к моменту, где я не нужен.»

Важность /init после настройки:
После успешной настройки нужно обязательно запустить /init с пояснением, чтобы Claude Code задокументировал рабочее состояние системы. Если сессия упадёт (watchdog не настроен), при перезапуске агент будет знать всю конфигурацию.
