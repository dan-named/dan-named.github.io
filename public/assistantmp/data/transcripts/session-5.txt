TickTick — настройка API-интеграции:
Катя использует TickTick как основной task-tracker. Настроили интеграцию по тому же принципу, что и Google Calendar: developer console → создание приложения → client ID и client secret → OAuth-авторизация. Столкнулись с проблемой redirect URL (вставили в app service URL вместо OAuth redirect URL) — после исправления авторизация прошла. /init — передача контекста между сессиями:
Дан показал команду /init в Claude Code — она создаёт файл CLAUDE.md с описанием текущей конфигурации (настроенные MCP-серверы, интеграции). Это способ передать контекст в следующую терминальную сессию, чтобы агент не «забывал» о настроенных интеграциях.

VPS на Digital Ocean:
Катя зарегистрировалась на Digital Ocean (sign in with Google). Создали дроплет: Ubuntu 24.04 LTS, 2GB RAM, базовый план за $12/мес, регион Сан-Франциско. Оплата через PayPal. Установили root-пароль для доступа. Подключились к серверу из терминала через SSH (без ключа, по паролю). VPS работает — Катя впервые подключилась к удалённому серверу.

Telegram-бот — архитектура:
Изначально идея была просто прокидывать заметки в Obsidian, но Катя сформулировала более амбициозную задачу: бот, через который можно управлять Google Calendar, создавать задачи, автоматизировать процессы. Для этого на VPS должна жить бесконечная сессия Claude Code с Telegram-плагином. Режим dangerously-skip-permissions позволяет агенту работать без подтверждений. Watchdog-скрипт каждые 2 минуты проверяет, жива ли сессия, и перезапускает при необходимости. У Дана такая система работает 5 дней без перезапуска.

Dropbox — мост между компьютером и VPS:
Для синхронизации Obsidian между локальным Mac и VPS нужен Dropbox. Папка Obsidian должна жить внутри папки Dropbox. Когда на VPS тоже будет установлен Dropbox с тем же аккаунтом — файлы синхронизируются автоматически. Катя начала установку Dropbox (basic free, до 2GB). Дан сгенерировал чек-лист полной настройки (60–90 минут) и скинул Кате для самостоятельной работы.

Daily notes в Obsidian:
Дан рассказал о своём формате: каждый день — отдельная daily-заметка. Когда отправляешь сообщение боту, он ищет заметку за сегодня, если нет — создаёт, если есть — дописывает в конец. Помимо daily — отдельные файлы под проекты и brainstorm.
